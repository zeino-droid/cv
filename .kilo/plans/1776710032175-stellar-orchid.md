# Plan d'amélioration - Générateur de CV

## Objectif
Améliorer la **qualité du contenu** généré dans les CV (prompts LLM, rewrites, structuration).

---

## 1. Amélioration des Prompts LLM

### 1.1 Refonte du prompt de génération (`prompts.py`)
- **Contexte enrichi** : Ajouter plus d'informations sur l'entreprise (taille, secteur) dans le prompt
- **Style guide** : Définir un guide de style explicite (ton, vocabulaire)
- **Exemplesfew-shot** : Ajouter 1-2 exemples de bonnes réponses dans le prompt
- **Contraintes renforcées** : Plus de règles sur le format (verbes d'action, évitant les répétitions)

### 1.2 Système de révision LLM
- **Auto-correction** : Faire relire le CV généré par le LLM pour validation
- **Score de qualité** : Ajouter un scoring (longueur, vocabulaire technique, impact)

---

## 2. Optimisation du Matching Expériences → CV

### 2.1 Matching sémantique avancé
- Remplacer le matching mot-clé par un scoring sémantique (embedding)
- Utiliser les descriptions d'expériences pour mieux les mapper aux jobs

### 2.2 Personnalisation par entreprise
- Adapter le ton du CV selon le type d'entreprise (startup vs grand compte)
- Modifier les mots-clés selon le secteur

---

## 3. Amélioration du Contenu Généré

### 3.1 Headlines optimisées
- Créer une bibliothèque de templates de headlines par profil
- Générer des headlines avec plus d'impact (chiffres, positionnement)

### 3.2 Bullet points meilleurs
- Contraindre davantage le format : `[Verbe] + [Impact] + [Métrique]`
- Ajouter une validation de la qualité des bullets (longueur, vocabulary)

### 3.3 Summary rewritting
- Résumer automatiquement le summary selon le poste
- Ajouter des "proof points" (chiffres clés) dans le summary

---

## 4. Tests & Validation

### 4.1 Métriques de qualité
- Longueur moyenne des sections
- Taux de mots techniques
- Score de lisibilité

### 4.2 Benchmark
- Comparer les CV générés vs CV originaux
- Feedback utilisateur sur la qualité

---

## Ordre de priorité

1. **Phase 1** (Rapide) : Few-shot examples + contraintes renforcées dans prompts
2. **Phase 2** (Moyen) : Système de révision/validation du contenu
3. **Phase 3** (Avancé) : Matching sémantique + templates de headlines
